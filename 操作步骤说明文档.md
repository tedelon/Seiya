# Seiya 操作步骤说明文档

## 概述

Seiya 是 pynsist 的增强版，提供以下核心功能：

1. **`.exe` 启动器**：使用 distlib launcher 机制生成独立 `.exe` 启动器（替代原 `.launch.pyw`）
2. **自动依赖解析**：只需列出直接依赖，传递依赖自动解析包含
3. **自定义 Logo**：支持 PNG、JPG、ICO、BMP、GIF、WEBP 等通用图片格式作为应用图标
4. **平台过滤**：自动排除 macOS/Linux 专用依赖
5. **元数据复制**：自动复制 `.dist-info` 目录，保证运行时元数据查询正常

## 一、环境准备

### 1.1 安装 Seiya

```bash
# 从 PyPI 安装
pip install seiya

# 或安装带图像格式支持的版本（JPG/BMP/GIF/WEBP）
pip install "seiya[image]"
```

### 1.2 从源码安装（开发模式）

```bash
git clone https://github.com/seiya-builder/seiya.git
cd seiya
pip install -e .
```

### 1.3 安装 NSIS

下载并安装 [NSIS](http://nsis.sourceforge.net/Download)。

### 1.4 可选：安装 Pillow（推荐）

Pillow 提供 JPG/BMP/GIF/WEBP 格式支持和更好的图标转换质量：

```bash
pip install Pillow
```

## 二、配置文件编写

### 2.1 最小配置示例

创建 `installer.cfg`：

```ini
[Application]
name=MyApp
version=1.0
entry_point=myapp:main

[Python]
version=3.12.10
bitness=64
format=bundled

[Include]
packages=flask
```

### 2.2 完整配置示例（含自定义 Logo）

```ini
[Application]
name=MyApp
version=1.0
publisher=My Company
entry_point=myapp:main
logo=mylogo.png
console=false
license_file=LICENSE.txt

[Python]
version=3.12.10
bitness=64
format=bundled
include_msvcrt=true

[Include]
packages=flask
  webview
pypi_wheels=
  pyqt5==5.15.6

[Build]
directory=build/myapp
installer_name=MyApp_Installer.exe

# 可选：额外快捷方式
[Shortcut Extra Tool]
entry_point=myapp:extra_func
logo=extra_logo.jpg
console=true

# 可选：命令行工具
[Command mycli]
entry_point=myapp:cli_func
console=true
```

### 2.3 配置项说明

#### [Application] 部分

| 选项 | 必填 | 说明 |
|------|------|------|
| `name` | 是 | 应用名称 |
| `version` | 是 | 应用版本 |
| `publisher` | 否 | 发布者名称 |
| `entry_point` | 否 | 入口点，格式 `模块:函数` |
| `script` | 否 | 启动脚本路径（与 entry_point 二选一） |
| `logo` | 否 | **自定义 Logo**，支持 PNG/JPG/ICO/BMP/GIF/WEBP 等格式 |
| `icon` | 否 | 与 `logo` 等效（`logo` 优先） |
| `console` | 否 | `true` 显示控制台窗口，默认 `false` |
| `license_file` | 否 | 许可证文件路径 |
| `extra_preamble` | 否 | 启动前执行的额外 Python 代码文件 |

#### [Python] 部分

| 选项 | 必填 | 说明 |
|------|------|------|
| `version` | 否 | Python 版本（需有嵌入式发行版，推荐 `3.12.10`） |
| `bitness` | 否 | 32 或 64，默认 64 |
| `format` | 否 | 固定为 `bundled` |
| `include_msvcrt` | 否 | 是否包含 MSVC 运行时，默认 `true` |

#### [Include] 部分

| 选项 | 必填 | 说明 |
|------|------|------|
| `packages` | 否 | **只需列出直接依赖**，传递依赖自动解析 |
| `pypi_wheels` | 否 | 从 PyPI 下载的 wheel 包列表 |
| `local_wheels` | 否 | 本地 wheel 文件路径 |
| `files` | 否 | 额外文件，格式 `源文件>目标目录` |
| `exclude` | 否 | 排除的文件模式 |

#### [Build] 部分

| 选项 | 必填 | 说明 |
|------|------|------|
| `directory` | 否 | 构建目录，默认 `build/nsis` |
| `installer_name` | 否 | 安装包文件名 |
| `nsi_template` | 否 | 自定义 NSI 模板 |

#### [Shortcut xxx] 部分（可选）

与 [Application] 相同，用于定义额外快捷方式。

#### [Command xxx] 部分（可选）

| 选项 | 必填 | 说明 |
|------|------|------|
| `entry_point` | 是 | 入口点 |
| `console` | 否 | 默认 `true` |
| `extra_preamble` | 否 | 额外前置代码 |

## 三、自定义 Logo 功能

### 3.1 支持的图片格式

| 格式 | 扩展名 | 是否需要 Pillow | 说明 |
|------|--------|-----------------|------|
| **ICO** | `.ico` | 否 | 直接使用，无需转换 |
| **PNG** | `.png` | 否（有则更优） | 自动转换为多尺寸 ICO |
| **JPEG** | `.jpg/.jpeg` | 是 | 需要 Pillow 解码 |
| **BMP** | `.bmp` | 是 | 需要 Pillow 解码 |
| **GIF** | `.gif` | 是 | 需要 Pillow 解码 |
| **WEBP** | `.webp` | 是 | 需要 Pillow 解码 |

### 3.2 使用方法

在配置文件中设置 `logo` 或 `icon` 选项：

```ini
[Application]
name=MyApp
version=1.0
entry_point=myapp:main
logo=mylogo.png        # 支持 PNG
# 或
logo=mylogo.jpg        # 支持 JPG
# 或
logo=mylogo.ico        # 支持 ICO
```

### 3.3 转换规则

1. **ICO 文件**：直接使用，不修改原文件
2. **PNG/JPG/其他格式**：自动转换为 ICO，生成 `原名_converted.ico`，**不修改原文件**
3. **转换质量**：使用 Pillow 时生成多尺寸 ICO（16/32/48/64/128/256），无 Pillow 时 PNG 包装为单尺寸 ICO

### 3.4 安装 Pillow（推荐）

```bash
pip install Pillow
```

Pillow 提供更好的图标转换质量，并支持 JPG 等格式。

## 四、自动依赖解析

### 4.1 工作原理

Seiya 会自动解析传递依赖：

1. 读取 `packages` 中列出的直接依赖
2. 通过 `importlib.metadata` 读取每个包的 `requires` 字段
3. 递归解析所有传递依赖
4. 通过 PEP 508 markers 过滤平台专用依赖（如 macOS 的 `pyobjc-*`）
5. 自动复制 `.dist-info` 元数据目录

### 4.2 示例

配置文件只需：
```ini
[Include]
packages=flask
  webview
```

自动解析的传递依赖（无需手动添加）：
- `flask` → `jinja2`, `werkzeug`, `click`, `itsdangerous`, `markupsafe`, `blinker`
- `webview` → `bottle`, `proxy_tools`, `pythonnet`, `clr_loader`, `cffi`, `typing_extensions`
- `cffi` → `pycparser`
- `pythonnet` → `clr`（顶级模块）

## 五、构建操作步骤

### 5.1 基本构建

```bash
# 进入项目目录
cd /path/to/myproject

# 构建安装包（三种方式任选其一）
seiya installer.cfg
# 或
python -m seiya installer.cfg
# 或（向后兼容）
pynsist installer.cfg
```

### 5.2 仅准备文件（不调用 NSIS）

用于调试，检查生成的文件结构：

```bash
seiya installer.cfg --no-makensis
```

### 5.3 完整构建流程

```bash
# 1. 清理旧构建（如有进程锁定先停止）
# Windows PowerShell:
Get-Process | Where-Object { $_.Name -match "MyApp|pythonw" } | Stop-Process -Force -ErrorAction SilentlyContinue
Remove-Item -Recurse -Force "build/nsis" -ErrorAction SilentlyContinue

# 2. 构建安装包
seiya installer.cfg

# 3. 生成的安装包位于
# build/nsis/<AppName>_<version>.exe
```

### 5.4 验证生成的文件

```powershell
# 检查 build 目录结构
Get-ChildItem "build/nsis" -Recurse -Depth 1 | Select-Object FullName

# 检查 pkgs 目录包含的依赖
Get-ChildItem "build/nsis/pkgs" -Directory | Select-Object Name

# 检查 .dist-info 元数据
Get-ChildItem "build/nsis/pkgs" -Directory -Filter "*.dist-info" | Select-Object Name
```

## 七、常见问题

### 7.1 图标相关

**Q: 报错 "invalid icon file"**

A: 原版 pynsist 不支持 PNG/JPG。请使用 Seiya，它会自动转换格式。确保 `logo=` 指向有效图片文件。

**Q: JPG 转换失败**

A: JPG 转换需要 Pillow。运行 `pip install Pillow` 安装。

**Q: 图标显示不正确**

A: 推荐使用 256x256 或更大的方形图片作为 Logo。PNG 带透明通道效果最佳。

### 7.2 依赖相关

**Q: 运行时报 `ModuleNotFoundError`**

A: 检查 `packages` 是否列出了所有直接依赖。传递依赖应自动解析，如未解析检查包的 metadata 是否完整。

**Q: 报 `PackageNotFoundError: No package metadata was found for xxx`**

A: Seiya 会自动复制 `.dist-info` 目录。如仍报错，检查包是否正确安装。

### 7.3 构建相关

**Q: `FileExistsError: build/nsis`**

A: 有进程锁定目录。停止相关进程后重试：
```powershell
Get-Process | Where-Object { $_.Name -match "AppName|pythonw" } | Stop-Process -Force
Remove-Item -Recurse -Force "build/nsis"
```

**Q: `BadZipFile` 错误**

A: Python 嵌入式包缓存损坏。Seiya 会自动检测并重新下载，如仍失败手动删除缓存：
```powershell
Remove-Item -Recurse -Force "$env:LOCALAPPDATA\pynsist\python-*.zip"
```

## 八、文件结构说明

构建后的 `build/nsis/` 目录结构：

```
build/nsis/
├── <AppName>.exe                    # 生成的启动器（distlib launcher）
├── <AppName>_<version>.exe          # NSIS 安装包
├── installer.nsi                    # NSIS 脚本
├── <logo>_converted.ico             # 转换后的 ICO 图标（如原图为 PNG/JPG）
├── Python/                          # 嵌入式 Python
│   ├── python.exe
│   ├── pythonw.exe
│   ├── python312._pth
│   └── ...
├── pkgs/                            # 所有 Python 包
│   ├── flask/
│   ├── jinja2/                      # 自动解析的传递依赖
│   ├── werkzeug/                    # 自动解析的传递依赖
│   ├── flask-3.0.0.dist-info/       # 元数据目录
│   └── ...
└── msvcrt/                          # MSVC 运行时（可选）
```

## 九、技术架构

### 9.1 .exe 启动器结构

生成的 `.exe` 启动器由三部分组成：

```
[distlib launcher exe] + [shebang] + [zip(__main__.py)]
```

1. **launcher exe**：从 distlib 获取的基础启动器（`t64.exe` 或 `w64.exe`）
2. **shebang**：`#!Python\pythonw.exe\r\n`，指定 Python 解释器路径（相对路径）
3. **zip**：包含 `__main__.py` 的 zip 文件，设置 sys.path 并调用入口函数

### 9.2 自动依赖解析流程

```
用户配置 packages=[flask, webview]
        ↓
resolve_dependencies()
        ↓
读取 importlib.metadata.distributions()
        ↓
递归解析 requires 字段
        ↓
PEP 508 marker 过滤（保留 Windows 兼容依赖）
        ↓
返回完整依赖列表
        ↓
copy_modules() + _copy_dist_info()
```

### 9.3 图标转换流程

```
logo=mylogo.png
        ↓
ensure_valid_icon()
        ↓
检测文件格式（ICO/PNG/JPG）
        ↓
├─ ICO: 直接使用
├─ PNG: png_to_ico()（Pillow 或包装）
└─ JPG: jpeg_to_ico()（需要 Pillow）
        ↓
生成 <name>_converted.ico
        ↓
复制到 build 目录 + 写入 NSI 脚本
```

## 十、与原版 pynsist 的兼容性

Seiya 完全兼容原版 pynsist 的所有功能：

- ✅ 所有配置文件格式（`[Application]`, `[Python]`, `[Include]`, `[Build]`, `[Shortcut]`, `[Command]`）
- ✅ `entry_point` 和 `script` 两种启动方式
- ✅ `pypi_wheels` 和 `local_wheels` 依赖获取
- ✅ 自定义 NSI 模板
- ✅ 额外文件包含（`extra_files`）
- ✅ 许可证文件
- ✅ 多快捷方式支持
- ✅ 命令行工具（`[Command]` 部分）
- ✅ `pynsist` 命令仍然可用（别名）

新增功能：
- 🆕 `seiya` 命令（独立入口，避免与原 pynsist 冲突）
- 🆕 `logo=` 配置选项（`icon` 的别名，支持更多格式）
- 🆕 PNG/JPG/BMP/GIF/WEBP 自动转 ICO
- 🆕 自动传递依赖解析
- 🆕 平台专用依赖自动过滤
- 🆕 `.dist-info` 元数据自动复制
- 🆕 `.exe` 启动器（替代 `.launch.pyw`）
